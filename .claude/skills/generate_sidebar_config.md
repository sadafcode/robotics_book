# SKILL: Generate Sidebar Configuration

## CONTEXT

The user needs to create or update the sidebar navigation for their Docusaurus documentation site. The sidebar determines how chapters and sections are organized and displayed in the navigation menu.

This skill handles:
- Auto-generated sidebars from directory structure
- Manual sidebar configuration with categories
- Multi-level hierarchical navigation
- Sidebar item ordering and grouping
- Category collapsing and labels

**Configuration request:** $ARGUMENTS (structure type, organization preferences)

## YOUR ROLE

Act as a documentation information architect with expertise in:
- Navigation design and user experience
- Content hierarchy and organization
- Docusaurus sidebar configuration
- Accessibility and discoverability patterns

## EXECUTION STEPS

### Step 1: Analyze Existing Content Structure

Scan the `docs/` directory to identify:
- All markdown files and their frontmatter
- Existing categories and grouping
- Current `sidebar_position` values
- Logical content relationships

```bash
# List all docs with frontmatter
find docs -name "*.md" -type f | sort
```

Parse each file's frontmatter to extract:
- `id`
- `title`
- `sidebar_label`
- `sidebar_position`
- Category membership (from file path)

### Step 2: Determine Sidebar Strategy

Choose the appropriate sidebar approach:

**Auto-generated (recommended for simple structures):**
- Docusaurus generates sidebar from file system and frontmatter
- Minimal configuration required
- Automatically updates when files are added/removed

**Manual configuration (for complex hierarchies):**
- Full control over organization and grouping
- Supports custom ordering and nesting
- Requires maintenance when structure changes

**Hybrid approach:**
- Mix of auto-generated sections and manual overrides
- Best for large documentation sites

Ask user preference if not specified in $ARGUMENTS.

### Step 3: Create Auto-Generated Sidebar

For auto-generated approach, configure `sidebars.js`:

```javascript
// @ts-check

/** @type {import('@docusaurus/plugin-content-docs').SidebarsConfig} */
const sidebars = {
  // Auto-generated sidebar from docs folder structure
  tutorialSidebar: [
    {
      type: 'autogenerated',
      dirName: '.', // Generate from docs folder
    },
  ],
};

module.exports = sidebars;
```

Ensure all markdown files have proper frontmatter:
- `sidebar_position`: numeric order
- `sidebar_label`: optional custom label

### Step 4: Create Manual Sidebar Configuration

For manual approach, build a structured configuration:

```javascript
// @ts-check

/** @type {import('@docusaurus/plugin-content-docs').SidebarsConfig} */
const sidebars = {
  tutorialSidebar: [
    // Single item
    'intro',

    // Category with items
    {
      type: 'category',
      label: 'Getting Started',
      collapsed: false,
      items: [
        'getting-started/installation',
        'getting-started/quick-start',
        'getting-started/configuration',
      ],
    },

    // Category with auto-generated items
    {
      type: 'category',
      label: 'Core Concepts',
      collapsed: false,
      items: [
        {
          type: 'autogenerated',
          dirName: 'core-concepts',
        },
      ],
    },

    // Nested categories
    {
      type: 'category',
      label: 'Advanced Topics',
      collapsed: true,
      items: [
        {
          type: 'category',
          label: 'Performance',
          items: [
            'advanced/performance/optimization',
            'advanced/performance/caching',
          ],
        },
        {
          type: 'category',
          label: 'Security',
          items: [
            'advanced/security/authentication',
            'advanced/security/authorization',
          ],
        },
      ],
    },

    // External link
    {
      type: 'link',
      label: 'GitHub Repository',
      href: 'https://github.com/username/repo',
    },
  ],
};

module.exports = sidebars;
```

### Step 5: Organize by Content Type

Structure sidebar based on documentation type:

**For tutorial-style books (linear progression):**
```javascript
tutorialSidebar: [
  'intro',
  'chapter-01-foundations',
  'chapter-02-basics',
  'chapter-03-intermediate',
  'chapter-04-advanced',
  'chapter-05-expert',
  'conclusion',
]
```

**For reference documentation (topic-based):**
```javascript
tutorialSidebar: [
  'intro',
  {
    type: 'category',
    label: 'API Reference',
    items: ['api/overview', 'api/authentication', 'api/endpoints'],
  },
  {
    type: 'category',
    label: 'Guides',
    items: ['guides/quick-start', 'guides/best-practices'],
  },
  {
    type: 'category',
    label: 'Examples',
    items: ['examples/basic', 'examples/advanced'],
  },
]
```

**For mixed content (hybrid approach):**
```javascript
tutorialSidebar: [
  'intro',
  {
    type: 'category',
    label: 'Part I: Fundamentals',
    collapsed: false,
    items: [
      {type: 'autogenerated', dirName: 'part-1'},
    ],
  },
  {
    type: 'category',
    label: 'Part II: Practical Applications',
    collapsed: false,
    items: [
      {type: 'autogenerated', dirName: 'part-2'},
    ],
  },
  {
    type: 'category',
    label: 'Resources',
    items: ['resources/glossary', 'resources/references'],
  },
]
```

### Step 6: Configure Multiple Sidebars (Optional)

For complex sites with distinct documentation sections:

```javascript
const sidebars = {
  // Main documentation sidebar
  docsSidebar: [
    'docs/intro',
    {
      type: 'category',
      label: 'Guides',
      items: [{type: 'autogenerated', dirName: 'docs/guides'}],
    },
  ],

  // API reference sidebar
  apiSidebar: [
    'api/overview',
    {
      type: 'category',
      label: 'Endpoints',
      items: [{type: 'autogenerated', dirName: 'api/endpoints'}],
    },
  ],
};
```

Update `docusaurus.config.js` to reference multiple sidebars:

```javascript
docs: {
  sidebarPath: './sidebars.js',
  routeBasePath: 'docs',
},
```

### Step 7: Validate Sidebar Configuration

Test the configuration:

```bash
# Build to check for errors
npm run build

# Start dev server to preview
npm run start
```

Check for:
- All document IDs resolve correctly
- Categories appear in intended order
- Collapsed/expanded states work
- No broken links or missing documents
- Sidebar renders on mobile devices

### Step 8: Optimize for User Experience

Apply UX best practices:

**Naming conventions:**
- Use clear, descriptive category labels
- Keep labels concise (2-4 words ideal)
- Use sentence case for consistency

**Hierarchy depth:**
- Avoid nesting more than 3 levels deep
- Group related items under meaningful categories
- Use auto-generation for flat sections

**Collapsed states:**
- Set `collapsed: false` for primary/getting-started sections
- Set `collapsed: true` for advanced/reference sections
- Consider user's likely entry points

**Ordering:**
- Place introductory content first
- Progress from basic to advanced
- Group by feature or topic, not by type

## ERROR HANDLING

**If document ID not found:**
- List all available document IDs
- Check for typos in sidebar config
- Verify file exists and has proper frontmatter

**If category is empty:**
- Check directory path is correct
- Ensure directory contains markdown files
- Verify frontmatter in contained files

**If build fails:**
- Display specific error message
- Identify problematic sidebar item
- Provide corrected configuration
