# SKILL: Add Module Structure

## CONTEXT

The user wants to add a new module, section, or content area to their existing Docusaurus documentation. This could be:

- A new chapter series or book section
- A new documentation module (API, Guides, Tutorials)
- A new category in the sidebar
- A reorganization of existing content
- Multi-version documentation support

**Module details:** $ARGUMENTS (module name, type, organization)

## YOUR ROLE

Act as a documentation architect with expertise in:
- Content information architecture
- Docusaurus plugin configuration
- Multi-module documentation organization
- Scalable documentation structures
- Content migration and reorganization

## EXECUTION STEPS

### Step 1: Analyze Module Requirements

Extract from $ARGUMENTS:
- **Module name**: Identifier for the new module (e.g., "api", "tutorials", "advanced")
- **Module type**: Category, versioned docs, plugin docs, or separate section
- **Content scope**: What will this module contain
- **Relationship to existing content**: Independent or nested
- **Navigation requirements**: Where in sidebar, how to access

### Step 2: Choose Module Structure Type

Determine the appropriate structure:

#### Type A: Category in Existing Sidebar
**Use when:** Adding a new section to current documentation

```
docs/
  ├── intro.md
  ├── existing-content/
  └── new-module/          ← New category
      ├── index.md
      ├── topic-1.md
      └── topic-2.md
```

#### Type B: Separate Documentation Plugin Instance
**Use when:** Adding distinct documentation (e.g., API docs separate from guides)

```
docs/              ← Main documentation
api/               ← Separate API docs
tutorials/         ← Separate tutorials
```

#### Type C: Versioned Documentation
**Use when:** Need to maintain multiple versions (e.g., v1.0, v2.0)

```
docs/              ← Current version
versioned_docs/
  └── version-1.0/
```

#### Type D: Nested Subcategories
**Use when:** Deep hierarchy needed within existing structure

```
docs/
  └── advanced/
      ├── performance/
      │   ├── caching.md
      │   └── optimization.md
      └── security/
          ├── auth.md
          └── encryption.md
```

### Step 3a: Add Category to Existing Sidebar

**Create directory structure:**

```bash
mkdir -p docs/$MODULE_NAME
touch docs/$MODULE_NAME/index.md
```

**Create index/overview page:**

`docs/$MODULE_NAME/index.md`:
```markdown
---
id: $MODULE_ID
title: $MODULE_TITLE
sidebar_label: Overview
sidebar_position: 1
---

# $MODULE_TITLE

$MODULE_DESCRIPTION

## What's Covered

- Topic 1
- Topic 2
- Topic 3

## Getting Started

Start with [Topic 1](./$TOPIC_1.md) to learn the fundamentals.
```

**Update sidebar configuration:**

In `sidebars.js`:
```javascript
module.exports = {
  tutorialSidebar: [
    'intro',
    // ... existing items
    {
      type: 'category',
      label: '$MODULE_LABEL',
      collapsed: false,
      items: [
        {
          type: 'autogenerated',
          dirName: '$MODULE_NAME',
        },
      ],
    },
  ],
};
```

### Step 3b: Add Separate Documentation Plugin Instance

**Create new documentation directory:**

```bash
mkdir -p $MODULE_NAME
mkdir -p $MODULE_NAME/docs
touch $MODULE_NAME/docs/intro.md
```

**Create sidebar for new module:**

`$MODULE_NAME/sidebars.js`:
```javascript
module.exports = {
  $MODULE_NAME_Sidebar: [
    'intro',
    {
      type: 'autogenerated',
      dirName: '.',
    },
  ],
};
```

**Configure in `docusaurus.config.js`:**

```javascript
module.exports = {
  // ... existing config

  presets: [
    [
      'classic',
      {
        docs: {
          // Main docs config
          sidebarPath: './sidebars.js',
          routeBasePath: 'docs',
        },
      },
    ],
  ],

  plugins: [
    [
      '@docusaurus/plugin-content-docs',
      {
        id: '$MODULE_ID',
        path: '$MODULE_NAME/docs',
        routeBasePath: '$MODULE_NAME',
        sidebarPath: './$MODULE_NAME/sidebars.js',
        editUrl: 'https://github.com/$USERNAME/$REPO/tree/main/',
      },
    ],
  ],

  themeConfig: {
    navbar: {
      items: [
        {
          type: 'docSidebar',
          sidebarId: 'tutorialSidebar',
          position: 'left',
          label: 'Docs',
        },
        {
          to: '/$MODULE_NAME/intro',
          label: '$MODULE_LABEL',
          position: 'left',
          activeBaseRegex: `/$MODULE_NAME/`,
        },
      ],
    },
  },
};
```

### Step 3c: Add Versioned Documentation

**Enable versioning:**

```bash
npm run docusaurus docs:version 1.0
```

This creates:
- `versioned_docs/version-1.0/` - Copy of current docs
- `versioned_sidebars/version-1.0-sidebars.json` - Sidebar snapshot
- Updates `versions.json`

**Configure version dropdown in navbar:**

In `docusaurus.config.js`:
```javascript
themeConfig: {
  navbar: {
    items: [
      {
        type: 'docsVersionDropdown',
        position: 'right',
        dropdownActiveClassDisabled: true,
      },
    ],
  },
},
```

**Create version-specific content:**

```bash
# Work in docs/ for next version
# Work in versioned_docs/version-1.0/ for v1.0 fixes
```

### Step 5: Set Up Module Navigation

**For navbar dropdown (multiple modules):**

```javascript
themeConfig: {
  navbar: {
    items: [
      {
        type: 'dropdown',
        label: 'Documentation',
        position: 'left',
        items: [
          {
            type: 'doc',
            docId: 'intro',
            label: 'Main Docs',
          },
          {
            to: '/$MODULE_NAME/intro',
            label: '$MODULE_LABEL',
          },
          {
            to: '/api/intro',
            label: 'API Reference',
          },
        ],
      },
    ],
  },
},
```

**For separate navbar items:**

```javascript
navbar: {
  items: [
    {to: '/docs/intro', label: 'Docs', position: 'left'},
    {to: '/$MODULE_NAME/intro', label: '$MODULE_LABEL', position: 'left'},
    {to: '/api/intro', label: 'API', position: 'left'},
  ],
},
```

### Step 6: Add Module-Specific Configuration

**Custom styling (optional):**

Create `$MODULE_NAME/custom.css`:
```css
/* Module-specific styles */
.$MODULE_NAME-container {
  /* custom styles */
}
```

Import in module pages:
```markdown
---
custom_edit_url: null
---

import './custom.css';
```

**Module metadata:**

Add to `docusaurus.config.js`:
```javascript
plugins: [
  [
    '@docusaurus/plugin-content-docs',
    {
      id: '$MODULE_ID',
      // ... other config
      editUrl: 'https://github.com/$USERNAME/$REPO/tree/main/$MODULE_NAME/',
      showLastUpdateTime: true,
      showLastUpdateAuthor: true,
    },
  ],
],
```

### Step 7: Migrate Existing Content (If Applicable)

If reorganizing existing content into new module:

**Create migration script:**

```bash
#!/bin/bash
# scripts/migrate-to-module.sh

MODULE_NAME=$1
SOURCE_DIR="docs"
TARGET_DIR="docs/$MODULE_NAME"

# Create target directory
mkdir -p "$TARGET_DIR"

# Move files (adjust patterns as needed)
mv "$SOURCE_DIR"/topic-*.md "$TARGET_DIR/"

# Update frontmatter sidebar_position
# (Manual step or use sed/awk)

echo "✅ Migrated files to $TARGET_DIR"
echo "⚠️  Update internal links to reflect new paths"
```

**Update internal links:**

Search and replace old paths with new paths:
```bash
# Example: update links in all markdown files
find docs -name "*.md" -type f -exec sed -i 's|(\./old-path/|(\./new-module/|g' {} +
```

## ERROR HANDLING

**If plugin configuration fails:**
- Check plugin ID is unique
- Verify path exists and is correct
- Ensure sidebars.js file is valid JavaScript
- Review docusaurus.config.js syntax

**If navbar item doesn't appear:**
- Verify route path matches module configuration
- Check `to` or `docId` is correct
- Ensure module has at least one document

**If versioning fails:**
- Check that docs/ directory exists and has content
- Verify versions.json is valid JSON
- Ensure no uncommitted changes in docs/
